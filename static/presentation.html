Feature Deep-Dive

The Live Weather Hero

The most powerful feature of ThermoML v2.0 — a real-time weather dashboard that also drives the prediction form.

Your City. Real Data. Right Now.

ThermoML uses the browser Geolocation API to get your coordinates, then reverse-geocodes them via OpenStreetMap Nominatim to show your city and country name — no sign-up, no API keys.

Live weather data streams in from Open-Meteo — a completely free, open-source weather API with worldwide coverage. This powers the full hero dashboard and auto-fills the ML prediction form.

ML Engine

Four Models, One App

ThermoML v2.0 ships with four independently trained scikit-learn models — each serialized as a .pkl file and loaded at startup by the Flask backend.

★ Best
🏆
Ensemble

Combines predictions from multiple base learners, reducing variance and bias. The recommended choice for most predictions — it averages out errors from individual models.

Highest Accuracy ★ Recommended
🚀
Gradient Boosting

Builds trees sequentially, each one correcting the residual errors of the last. Excellent for structured/tabular data with complex non-linear relationships.

Non-linear Patterns Boosting
🌲
Random Forest

An ensemble of decision trees trained on random subsets of data and features. Robust against overfitting and great at capturing feature interactions.

Robust to Noise Bagging
📈
Linear Regression

The interpretable baseline — finds the best-fit linear relationship between weather features and temperature. Fast, transparent, and useful as a benchmark.

Baseline / Interpretable Fast

Project Journey

From Data to Deployment

The full lifecycle of building ThermoML — from raw dataset to a live, browser-based ML application.

Phase 01 · Data
Dataset Collection & Feature Engineering
Assembled a weather dataset covering temperature, humidity, wind speed, pressure, rainfall, sunlight hours, and day-of-year. Cleaned nulls, normalized features, and engineered day-of-year as a cyclical seasonal signal.
Phase 02 · Training
Training Four scikit-learn Models
Trained Linear Regression, Random Forest, Gradient Boosting, and a custom Ensemble model in Python. Serialized all four as .pkl files using joblib for fast Flask serving. Evaluated and compared each model's MAE, RMSE, and R² scores.
Phase 03 · Backend
Flask REST API
Built a clean Flask app.py that loads all four model files at startup, validates input, handles missing/bad fields gracefully, and returns predictions with model labels. Deployed on 0.0.0.0 with debug mode for development.
Phase 04 · Frontend v1
Initial Weather Prediction Form
First version featured a simple HTML form with 2-column layout, DM Sans typography, and basic dark blue palette. One model (Linear Regression) available. Functional but visually plain.
Phase 05 · Frontend v2.0
ThermoML — Full Design Overhaul
Complete rebrand to ThermoML. New color system (Electric Blue, Neon Cyan, Warm Amber), Fraunces + JetBrains Mono + Outfit font stack, dot-grid background, glow system, and a completely redesigned component library — model tab cards, input fields, result box, and footer.
Phase 06 · New Feature
Live Weather Hero + Smart Auto-Fill
Added the live weather hero section using Open-Meteo (free, no key) and Nominatim for reverse geocoding. Browser geolocation drives real-time city display, weather stats, 12-hour forecast strip, and automatic form field population — the app's signature feature.

v1 → v2.0 Upgrades

Every major improvement shipped in ThermoML v2.0 over the original version.

Live Weather Integration

Entirely new feature. The app now fetches real-time weather data using your browser's geolocation — something v1 had no concept of.

Live weather values auto-populate the prediction form. In v1, users had to manually research and enter every single value.

Mobile-first responsive layout at all breakpoints. v1 was desktop-only. ThermoML v2.0 works seamlessly on phones and tablets.

Staggered page load animations, pulsing live indicators, hover micro-interactions, and smooth transitions — all new in v2.0.
